method och kommunikation protokol tcp och pipe samma

    req:
        * authentication key ("auth-key")

        * requests' method ("method"):
            1) "GET": to get data
            2) "POST": to share data with server e.g. insert data into server's database
        * requests' ID ("reqID")
        * request's body ("body"): if type of request is "POST" then it should have even a body

        request schema:
            {   "title": "request schema",
                "description": "format a valid request ",
                "type": "object",
                "properties": {
                    "auth-key": { "type": "string"},
                    "method": { "type": "string"},
                    "reqId": { "type": "string"},
                    "body": { "type": "object"}
                }
                "required": ["auth-key", "method", "reqId", "body"]
            }

    response:
        * "reqId": indicates which request the response belongs to
        * "body": the actual response for a GET request, "complete" för a POST request or "404 bad request"
        response:
        {   "title": "response schema"
            "description": "format a valid response ",
            "type": "object",
            "properties": {
                    "reqId": { "type": "string"},
                    "body": { "type": "object"}
            }
            "required": ["reqId", "body"]
        }
